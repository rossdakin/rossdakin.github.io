Real Targets

  • clean
  • deps: generates deps.js file for on-the-fly source loading
  • docs: generated JSDoc
  • manifest: generates list of all files to include in build, in dependency order
  • source-map: generates a source map
  • source-map-assets: copies relevant source code to the output directory
  • soy: generates JavaScript from .soy template files
  • test: runs tests
  • type-check: runs static type checks
  • update: updates libraries used

Summary

Drawbacks

  • No real-time updating (e.g. Plovr)
  • Assumes use of Closure Library
  • No built-in versioning
  • No built-in repo support (e.g. tag upon build)
  • No built-in i18n in templates
  • Docs are a bit hacky (must generate JSDoc tags for parent namespaces)
  • Type checking is slow/hacky (must check entire Closure Library)

Summary

Benefits

  • Lightweight
  • Common prereq's
  • Fairly comprehensive
